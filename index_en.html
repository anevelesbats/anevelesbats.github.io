Radio Communication System Simulation

ENSEIRB–MATMECA: Second-year project

Project goals

The project aims to introduce the simulation of a telecommunications transmit–receive chain implementing QPSK modulation.
The chain is made up of predefined RF components (amplifiers, mixers, modulators, filters, oscillators, antennas, propagation channel…)

Concepts used

  • QPSK (Quadrature Phase Shift Keying)
  • Raised-cosine filter
  • RF transmit–receive chain
  • RF/IF mixer
  • Power amplifier (PA) and low-noise amplifier (LNA)
  • Non-linearity and 1 dB compression point (P1dB)
  • GSM channel (loss, attenuation, phase shift, delay)
  • Constellation and eye diagram
  • Bit Error Rate (BER)

Command-line Media Library

ENSEIRB–MATMECA: Second-year project

Project goals

The goal is to develop a C++ application to manage a media library, applying object-oriented programming principles. The application must provide features such as adding, deleting, consulting and searching resources (books, CDs, DVDs, journals, etc.).
It must also support saving/loading data, and managing loans and reservations.
The project includes an extension for multi-user management with differentiated rights for administrators and clients.

Concepts used

  • Classes and objects: user-defined types.
  • Encapsulation (public / private / protected): interface–implementation separation.
  • Inheritance: derived classes.
  • Polymorphism and virtual functions: dynamic dispatch via base pointers/references.
  • Abstract classes and pure virtual methods: shared non-instantiable base.
  • Constructors and destructors — object initialization and cleanup.
  • Header/source separation: .hpp for interfaces and .cpp for implementations.
  • Pointers and references: managing relationships between objects (passing and storage).
  • STL containers: storing collections (resources, users).

GitHub repository

Processor Design with Instruction Set

ENSEIRB–MATMECA: Second-year project

Project goals

The objective is to design a simple processor in VHDL with a basic instruction set. The processor must be able to execute arithmetic, logic and control-flow instructions. The project includes the design of the control unit, the Arithmetic and Logic Unit (ALU), registers and memory.
A test interface must also be developed to simulate and validate the processor’s behavior.

Concepts used

  • Hardware design in VHDL: hardware description.
  • Processor architecture: components and organization.
  • Instruction set: supported operations.
  • Control unit: instruction flow management.
  • ALU: arithmetic and logical operations.
  • Registers and memory: data and instruction storage.
  • Simulation and testing: validation of functionality.
