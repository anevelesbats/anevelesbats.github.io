Simulation of a Radio Communication System

Enseirb-Matmeca: Second-year project

Project Objectives

The project introduced students to the simulation of a telecommunication transmission–reception system using QPSK modulation. The chain consisted of predefined RF components (amplifiers, mixers, modulators, filters, oscillators, antennas, propagation channel, etc.).

Concepts Used

  • QPSK Modulation (Quadrature Phase Shift Keying)
  • Raised Cosine Filter
  • RF Transmit–Receive Chain
  • RF/IF Mixer
  • Power Amplifier (PA) and Low-Noise Amplifier (LNA)
  • Non-linearity and 1 dB Compression Point (P1dB)
  • GSM Propagation Channel (loss, attenuation, phase shift, delay)
  • Constellation and Eye Diagram
  • Bit Error Rate (BER)

Command-Line Media Library

Enseirb-Matmeca: Second-year project

Project Objectives

The goal of this project was to develop a C++ application for managing a media library, applying object-oriented programming principles. The application provides features such as adding, deleting, browsing, and searching for resources (books, CDs, DVDs, magazines, etc.), as well as saving and loading data and managing borrowing and reservation operations. It also supports multi-user management with distinct permissions for administrators and clients.

Concepts Used

  • Classes and objects: type definition.
  • Encapsulation (public / private / protected): interface vs implementation separation.
  • Inheritance: derived classes.
  • Polymorphism and virtual functions: dynamic calls via base pointers and references.
  • Abstract classes and pure virtual methods: non-instantiable base classes.
  • Constructors and destructors: object initialization and cleanup.
  • Header/source separation: .hpp for interfaces, .cpp for implementations.
  • Pointers and references: managing relationships between objects (passing and storage).
  • STL containers: storing collections (resources, users).

Processor Design with Instruction Set

Enseirb-Matmeca: Second-year project

Project Objectives

The goal was to design a simple processor in VHDL with a basic instruction set. The processor performs arithmetic, logical, and control flow operations. The project includes the design of the control unit, ALU, registers, and memory, as well as a test interface to simulate and validate its operation.

Concepts Used

  • Hardware design in VHDL.
  • Processor architecture: components and organization.
  • Instruction set: supported operations.
  • Control unit: instruction flow management.
  • Arithmetic Logic Unit (ALU): arithmetic and logic operations.
  • Registers and memory: data and instruction storage.
  • Simulation and testing: validation of functionality.
